# GitHub Profile Analyzer 사용 가이드

## 빠른 시작

### 1. 실행하기
```bash
./run.sh
```

### 2. 메뉴 선택
```
Main Menu
1 Start Analysis
0 Exit

Select option [1]: 1
```

### 3. 사용자명 입력
```
GitHub Username: octocat
```

또는 URL 전체를 입력해도 됩니다:
```
GitHub Username: https://github.com/octocat
```

### 4. 리포지토리 타입 선택
```
Repository Type Selection:
1. Public only      (공개 리포지토리만)
2. Private only     (비공개 리포지토리만)
3. Both (default)   (둘 다 분석 - 기본값)

Select type [3]: 3
```

- **Public only**: 공개 리포지토리만 분석
- **Private only**: 비공개 리포지토리만 분석 (Token에 repo 권한 필요)
- **Both**: 모든 리포지토리 분석 (기본값, 추천)

### 5. Token 입력
```
GitHub Personal Access Token Required
Token will be used immediately and discarded

Enter Token (hidden): [입력 - 화면에 표시되지 않음]
```

**중요**: 
- Token은 화면에 표시되지 않습니다
- Token은 즉시 사용되고 메모리에서 제거됩니다
- 컴퓨터에 저장되지 않습니다

### 6. 분석 진행
```
Starting analysis for: octocat
Type: ALL

Token discarded from memory

Analyzing repositories... ████████████ 100%
```

### 7. 결과 확인

## Token 발급 방법

### GitHub에서 Token 생성

1. **GitHub 로그인** → https://github.com

2. **Settings 이동**
   - 우측 상단 프로필 클릭
   - Settings 클릭

3. **Developer settings**
   - 왼쪽 메뉴 맨 아래
   - Developer settings 클릭

4. **Personal access tokens**
   - Personal access tokens 클릭
   - Tokens (classic) 선택

5. **Generate new token**
   - "Generate new token (classic)" 클릭

6. **권한 설정**
   
   **필수 권한**:
   - `repo` (전체) - Private 리포지토리 접근용
     - repo:status
     - repo_deployment
     - public_repo
     - repo:invite
     - security_events
   - `user` (읽기) - 사용자 정보 접근용
     - read:user
     - user:email

   **Public만 분석할 경우**:
   - `public_repo` 만 선택해도 됩니다

7. **Token 생성**
   - "Generate token" 클릭
   - **⚠️ Token을 즉시 복사하세요!** (다시 볼 수 없습니다)

8. **Token 사용**
   - 프로그램 실행 시 입력
   - 파일로 저장하지 마세요

## 생성된 파일 설명

### JSON 파일 (`{username}_analysis.json`)

전체 분석 데이터가 포함된 JSON 파일:

```json
{
  "username": "octocat",
  "repo_type": "all",
  "repositories": [
    {
      "name": "Hello-World",
      "url": "https://github.com/octocat/Hello-World",
      "private": false,
      "commits": 123,
      "stars": 456,
      "language": "JavaScript",
      "created_at": "2023-01-01T00:00:00Z",
      ...
    }
  ],
  "statistics": {
    "total_repositories": 42,
    "total_commits": 1234,
    "language_distribution": {...},
    ...
  }
}
```

**용도**: 
- 데이터 백업
- 다른 도구에서 재사용
- 프로그래밍 방식 분석

### HTML 파일 (`{username}_visualization.html`)

인터랙티브 시각화 리포트:

**포함 내용**:
- 📊 Chart.js 기반 그래프 5개
- 📋 상세 리포지토리 테이블
- 📈 통계 카드
- 🖼️ Export 기능

**사용 방법**:
```bash
# 브라우저로 열기
xdg-open octocat_visualization.html

# 또는 직접 브라우저에서 파일 열기
```

**Export 기능**:
1. **Export as Image**: 전체 리포트를 PNG로 저장
2. **Export Table as Image**: 테이블만 PNG로 저장
3. **Print / Save as PDF**: 브라우저 인쇄 기능으로 PDF 저장

## 고급 사용법

### 여러 사용자 분석

```bash
./run.sh
# 첫 번째 사용자 분석
# user1_analysis.json, user1_visualization.html 생성

./run.sh
# 두 번째 사용자 분석
# user2_analysis.json, user2_visualization.html 생성
```

### 같은 사용자 재분석

같은 사용자를 다시 분석하면 자동으로 번호가 추가됩니다:

```
octocat_analysis.json       (첫 번째)
octocat_analysis_1.json     (두 번째)
octocat_analysis_2.json     (세 번째)
```

### Private 리포지토리만 분석

자신의 비공개 프로젝트만 분석하고 싶을 때:

```bash
./run.sh
# Username: yourname
# Type: 2 (Private only)
# Token 입력 (repo 권한 필요)
```

### Public 리포지토리만 분석

타인의 공개 프로젝트를 분석할 때:

```bash
./run.sh
# Username: torvalds
# Type: 1 (Public only)
# Token 입력 (public_repo 권한만 필요)
```

## 문제 해결

### Rate Limit 초과

**증상**:
```
Error: API rate limit exceeded
```

**해결**:
- Token을 사용하면 시간당 5,000건으로 증가
- 1시간 후 다시 시도
- GitHub Settings에서 rate limit 확인:
  https://github.com/settings/applications

### Token 권한 부족

**증상**:
```
Error: Resource not accessible by token
```

**해결**:
1. Token 재생성
2. `repo` 전체 권한 부여
3. Private 리포지토리 접근 시 필수

### 파일이 생성되지 않음

**확인**:
```bash
# 현재 디렉토리 확인
pwd

# 생성된 파일 확인
ls -la *_analysis*.json *_visualization*.html
```

**해결**:
- 프로그램을 실행한 디렉토리에 파일이 생성됩니다
- 쓰기 권한 확인

### HTML 차트가 표시되지 않음

**원인**: 
- 인터넷 연결 필요 (Chart.js CDN 사용)

**해결**:
- 인터넷 연결 확인
- 브라우저 JavaScript 활성화 확인

### 한글이 깨짐

**HTML 파일**:
- UTF-8 인코딩 자동 적용
- 최신 브라우저 사용

**터미널**:
```bash
# 로케일 확인
locale

# UTF-8 설정
export LANG=ko_KR.UTF-8
```

## 보안 주의사항

### ✅ 안전한 사용법

1. **Token 직접 입력**: 프로그램 실행 시에만 입력
2. **파일로 저장하지 않기**: .env, .txt 등에 저장 금지
3. **화면 공유 조심**: Token 입력 시 화면에 표시되지 않음
4. **프로그램 신뢰**: 오픈소스, GitHub에 공개됨

### ❌ 위험한 행동

1. **Token을 파일로 저장**: 절대 금지
2. **Token을 Git에 커밋**: .gitignore에 차단됨
3. **Token 공유**: 다른 사람에게 절대 공유하지 마세요
4. **Token 스크린샷**: 화면 캡처 시 Token 노출 주의

### 🔒 프로그램의 보안 기능

1. **즉시 폐기**: Token을 받자마자 사용하고 메모리에서 제거
   ```python
   del token
   gc.collect()
   ```

2. **파일 저장 안 함**: Token이 어떤 파일에도 저장되지 않음

3. **Git 추적 제외**: `.gitignore`에 Token 관련 모든 파일 차단

4. **메모리 정리**: 분석 완료 후 모든 객체 제거

## 생성된 데이터 관리

### 파일 정리

```bash

ls -la *_analysis*.json *_visualization*.html

# 특정 사용자 파일만 삭제
rm octocat_*

rm *_analysis*.json *_visualization*.html
```

### Git 추적

생성된 파일은 자동으로 Git 추적에서 제외됩니다:
```
.gitignore:
*.json
*.html
*.png
```

공유하고 싶다면 `.gitignore`를 수정하세요.
